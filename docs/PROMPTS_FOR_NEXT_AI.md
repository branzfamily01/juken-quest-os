## ① 教材JSON変換プロンプト(いちばん使う・錯乱肢ルール入り)

添付・貼り付けるもの:教材の内容(テキスト・写真の書き起こし・範囲表)

```
あなたは中学受験の教材編集者です。以下の教材内容を、学習アプリ用の
「教材JSON(juken-pack@1)」に変換してください。

# 出力ルール(厳守)
- JSONのみを出力。前置き・説明・マークダウンの```は不要
- schema は "juken-pack@1" 固定。id は "pack_" + 教科略号(m/j/r/s) + 学年 + 内容
- subject は「算数」「国語」「理科」「社会」のいずれか

# ユニット構成(この順で)
1. INPUT: 重要知識のカード(front=問いかけ, back=答え。10〜20枚)
2. TRAIN: 練習問題 8〜15問。qtype をバランスよく混ぜる:
   - choice(4択): 5割程度
   - truefalse(○×): 2割程度。「まぎらわしい言い換え」で出す
   - multi(すべて選べ): 1〜2問。分類問題(「中性のものをすべて」等)に
   - order(並べかえ): 1〜2問。年代順・手順・大小順があれば必ず入れる
   - flashcard(暗記カード): 1〜2問。「声に出して言おう」と促す文言で
   - input(記入): 最大2問まで。計算結果の数字など必要な時だけ
3. TEST: 本番形式 5〜10問(timeLimit=60、choiceとinput中心、hints不要)
4. BOSS: 総まとめ4〜6問(重要問題の類題、bossName付き)
5. WRITE: 手書きが必要な内容(記述・作図・漢字・計算過程)があれば
   paperTask に instructions と checkPoints(3〜4個)

# 錯乱肢ルール(最重要)
選択肢の誤答は、無関係な語ではなく「本当に覚えていないと選べない」ものにする。
各choice/multi問題の誤答には、次のいずれかを必ず1つ以上含めること:
- 同じカテゴリの紛らわしい別項目(例:頼朝の誤答に「義経」「清盛」)
- ありがちな計算ミス・勘違いの結果(例:×0.3を÷0.3した値)
- 一字違い・同音・似た言い回し(例:「口が重い」vs「口がかたい」)
- 逆の意味・反対の操作
消去法で当たる「明らかに変な選択肢」は禁止。
truefalseも「惜しい言い換え」(例:正しい文の主語だけ入れ替える)で作る。

# 問題フィールドの書き方
- choice: answer は正解の位置(先頭に正解を書いてよい。アプリ側でシャッフルされる)
- multi: answers は正解位置の配列
- order: items に正しい順で書く(アプリ側でシャッフルされる)
- flashcard: front / back(text不要)
- 各問に explanation(小学生に分かる言葉。なぜ誤答がダメかにも触れる)を必ず
- TRAINの問題には hints を2段階(方向づけ→ほぼ答え)
- tag は「大分類:小分類」形式で統一(例「割合:食塩水」「歴史:鎌倉」)
- level は 1(基本)〜5(難問)

# 対象・難易度
- 対象学年: 【小◯年】(小3以下なら漢字にひらがなを多めに)
- 難易度の目安: 【組分け・合不合・全統小・◯◯中過去問 など】

# 教材内容
【ここに教材テキスト・写真の書き起こし・範囲表を貼る】
```

できたJSONは、アプリの「データ」画面 →「JSONを貼り付けて登録」に貼ります。

## ③ 新出題形式の追加プロンプト

添付するもの:本体HTML と AI_HANDOFF.md と 統合仕様書

```
(⓪の開始テンプレを先頭に貼ったうえで)

今回の依頼:
新しい出題形式「【例:fillblank(穴埋め)】」を追加してください。
AI_HANDOFF.md §4-B の手順(FMT_NAMES→renderQ分岐→ans関数→judge接続→readQ対応)に従うこと。

形式の仕様:
- 【どんな画面で、子どもが何をタップして答えるかを書く】
- 【JSONの書き方の希望(フィールド名など)があれば書く】
- タップ中心・スマホで押しやすい大きさにすること
- 採点は必ず既存の judge(true/false) を呼ぶこと(成績処理を自作しない)

完了後、統合仕様書に追記すべき「出題形式スキーマ」の文面と、
①教材変換プロンプトに追記する説明文も一緒に出してください。
```

### 使い方のコツ

- **1回の依頼に1テーマ**。「テーマ追加と新形式を同時に」は事故のもと
- 修正版を受け取ったら、まず**古いHTMLを残したまま**新しい方を開いて動作確認
- 動いたら AI_HANDOFF.md §6 の最短コース(ウィザード→教材登録→TRAIN→ふくしゅう→バックアップ→リロード)を自分でも一周
- うまくいったら、修正版HTMLを新しい「正」として保管し、日付をつけてバックアップ
