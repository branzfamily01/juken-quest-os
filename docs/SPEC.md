## 1. まず一言でいうと

受験クエストOSは、

> **1つのHTMLアプリに、教材JSONを貼り付けるだけで、教材をゲーム化できる学習アプリ**

です。

コードを読む必要はありません。
使う人が触るのは、基本的に次の3つだけです。

1. **HTMLファイル**  
   アプリ本体。ダブルクリックまたはブラウザで開く。

2. **教材JSON**  
   ChatGPTやClaudeで作る教材データ。アプリの「データ」画面に貼る。

3. **バックアップJSON**  
   成績・プロフィール・教材をまとめて保存する控え。

---

## 2. たとえで理解する

| たとえ | 受験クエストOSでの名前 | 役割 |
|---|---|---|
| ゲーム機本体 | HTMLファイル | アプリそのもの |
| ゲームソフト | 教材JSON | 算数・国語・理科・社会の教材 |
| セーブデータ | バックアップJSON | 成績・プロフィール・追加教材の保存 |
| 着せ替え | テーマJSON | 色・キャラ・ガチャの見た目 |
| 初期設定カード | config JSON | play / custom / platform などの版を切り替える |

つまり、HTMLを毎回作り直すのではなく、**教材JSONを差し替えて使い回す**考え方です。

## 3. v2で何が変わったか

| 項目 | v1 | v2最新版 |
|---|---|---|
| 問題形式 | 4択・記入中心 | 6形式に拡張 |
| 4択の順番 | 固定 | 毎回シャッフル |
| プロフィール | 固定 | セットアップウィザードで作成 |
| テーマ | 2種類 | 10種類から選択 |
| 出題設定 | なし | 親が範囲を決め、子が今日の設定を選ぶ |
| 音声 | なし | 問題読み上げ・聞き流し対応 |
| バックアップ | テキスト書き出し中心 | ファイルDL・3日リマインド対応 |
| 販売対応 | なし | play / custom / platform の3版に分岐 |
| リセット | 全消去のみ | 成績リセット・プロフィール削除・工場出荷を区別 |

## 4. 実際に使うときの流れ

### A. 教材を作る

ChatGPTやClaudeに、教材内容を渡して、`juken-pack@1` の教材JSONを作らせます。

例：

- 予習シリーズの範囲
- 組分けテスト範囲
- 週テスト対策
- 間違えた問題の復習
- 白地図・漢字・理科用語など

### B. アプリに入れる

1. `index.html` を開く
2. 下メニューの **「⚙️ データ」** を押す
3. **「📚 JSONを貼り付けて登録」** に教材JSONを貼る
4. **「登録する」** を押す
5. **「📚 クエスト」** から教材を選ぶ

### C. 子どもが学習する

- INPUT：知識カード
- TRAIN：練習
- TEST：本番形式
- BOSS：総まとめ
- WRITE：紙で手書き
- REVIEW：間違い復習

---

## 5. 教材JSONの基本構造

教材JSONは、次の順番で作ります。

```text
教材パック
├─ INPUT   知識カード
├─ TRAIN   練習問題
├─ TEST    本番形式
├─ BOSS    総まとめボス
└─ WRITE   紙でやる課題
```

### 必ず守る基本項目

| 項目 | 意味 | 例 |
|---|---|---|
| schema | 教材の種類 | `juken-pack@1` |
| id | 教材の識別番号 | `pack_s5_chihou_11_19` |
| title | 教材名 | `社会5年上 第11〜19回まとめ` |
| subject | 教科 | `算数` / `国語` / `理科` / `社会` |
| grade | 学年 | `5` |
| units | 中身 | INPUT / TRAIN / TEST / BOSS / WRITE |

## 6. v2で使える6つの問題形式

### 1. choice：4択

```json
{
  "qtype": "choice",
  "text": "日本最大の湖はどれですか。",
  "choices": ["琵琶湖", "霞ヶ浦", "十和田湖", "宍道湖"],
  "answer": 0
}
```

**ポイント**

- `answer` は、正解が `choices` の何番目かを表す数字です。
- 先頭は `0` です。
- 正解を先頭に置いてOKです。アプリ側でシャッフルされます。

---

### 2. truefalse：○×

```json
{
  "qtype": "truefalse",
  "text": "やませは東北地方に冷害をもたらすことがある。",
  "answer": true
}
```

**ポイント**

- 正しい文なら `true`
- 間違った文なら `false`
- ただし、雑な文ではなく「惜しい言い換え」にします。

---

### 3. multi：すべて選べ

```json
{
  "qtype": "multi",
  "text": "広島県にある世界遺産をすべて選びなさい。",
  "choices": ["原爆ドーム", "厳島神社", "石見銀山", "姫路城"],
  "answers": [0, 1]
}
```

**ポイント**

- 正解が複数あるときに使います。
- `answers` に正解の番号を並べます。

---

### 4. order：並べかえ

```json
{
  "qtype": "order",
  "text": "南から北へ並べなさい。",
  "items": ["九州", "中国地方", "近畿地方", "中部地方", "関東地方", "東北地方", "北海道"]
}
```

**ポイント**

- `items` には、正しい順で書きます。
- アプリ側でシャッフルされます。

---

### 5. flashcard：暗記カード

```json
{
  "qtype": "flashcard",
  "front": "声に出して言おう：東北地方6県の県庁所在地は？",
  "back": "青森市・盛岡市・秋田市・山形市・仙台市・福島市"
}
```

**ポイント**

- 子どもが「覚えていた / まだ」を自己申告します。
- 声に出す暗記に向いています。

---

### 6. input：記入

```json
{
  "qtype": "input",
  "text": "2万5千分の1の地形図で、地図上4cmは実際には何kmですか。数字だけで答えなさい。",
  "answer": "1",
  "accept": ["1km", "１", "１km"]
}
```

**重要**

- 別解は `accept` に書きます。
- `acceptedAnswers` ではなく、**`accept`** に統一します。
- 記入問題は打ち込みが面倒なので、最大2問程度に抑えるのがよいです。

## 7. 問題作成で絶対に大事なこと

### 誤答は「本当に迷うもの」にする

悪い例：

```text
日本最大の湖は？
A 琵琶湖
B アマゾン川
C 富士山
D 太平洋
```

これは簡単すぎます。

良い例：

```text
日本最大の湖は？
A 琵琶湖
B 霞ヶ浦
C 十和田湖
D 宍道湖
```

このように、**同じカテゴリのまぎらわしいもの**を入れます。

### 誤答に入れるべきもの

- 同じカテゴリのまぎらわしい別項目
- ありがちな計算ミス
- 一字違い・同音異義語
- 逆の意味・反対操作

### truefalseも雑にしない

悪い例：

```text
琵琶湖はアメリカにある。
```

良い例：

```text
琵琶湖は京都府にある。
```

正解は「×」。滋賀県と京都府を混同させる、受験らしい問題になります。

---

## 8. TRAIN / TEST / BOSS の違い

| 種類 | 役割 | ヒント | 解説 | 使いどころ |
|---|---|---|---|---|
| TRAIN | 練習 | あり | あり | 普段の確認 |
| TEST | 本番形式 | なし | あり | 組分け前の確認 |
| BOSS | 総まとめ | なし | あり | 単元のラスボス |

TRAINには `hints` を2つ入れます。

```json
"hints": ["まず地方名を考えよう", "瀬戸内海沿岸です"]
```

TESTとBOSSは、本番感を出すため、ヒントなしでよいです。

---

## 9. WRITE：紙でやる課題

社会・理科・算数・国語には、アプリだけでは不十分な内容があります。

例：

- 白地図に書き込む
- 漢字を書く
- 計算過程を書く
- 作図する
- 記述答案を書く

その場合は、WRITEを入れます。

```json
{
  "type": "WRITE",
  "title": "白地図で確認",
  "paperTask": {
    "instructions": "九州地方の県名と県庁所在地を白地図に書き込みましょう。",
    "checkPoints": [
      "県名を書いた",
      "県庁所在地を書いた",
      "主な山地・川・平野を書いた"
    ]
  }
}
```

## 10. custom版にする方法

教材を自分で追加するには、**custom版**にします。

| 版 | 教材追加 | テーマ追加 | アプリ名変更 | 用途 |
|---|---:|---:|---:|---|
| play | × | × | × | 同梱教材だけで遊ぶ |
| custom | ○ | ○ | × | 家庭で教材を追加する |
| platform | ○ | ○ | ○ | 塾・事業者向け |

### コードを触らない方法

アプリの **「⚙️ データ」→「📚 JSONを貼り付けて登録」** に、次の設定JSONを貼ります。

```json
{
  "schema": "juken-config@1",
  "edition": "custom",
  "brand": {
    "title": "受験クエストOS",
    "logoEmoji": "🗺️"
  },
  "flags": {
    "canAddMaterial": true,
    "canEditTheme": true,
    "canBrand": false,
    "gachaEnabled": true
  },
  "balance": {
    "xpRate": 1.0,
    "gachaRates": {
      "N": 70,
      "R": 22,
      "SR": 6,
      "SSR": 2
    },
    "dailyGoalDefault": 10
  },
  "parentRange": {
    "levels": [1, 2, 3, 4, 5],
    "counts": [5, 10, 15, 20],
    "formats": ["choice", "truefalse", "multi", "order", "flashcard", "input"]
  }
}
```

登録後、データ画面の上部に **「エディション：おうちカスタム版」** と出ればOKです。

## 11. 教材JSONを入れる場所

教材JSONは、必ずここに入れます。

```text
⚙️ データ
↓
📚 JSONを貼り付けて登録
↓
登録する
```

### 入れてはいけない場所

```text
💾 バックアップ欄
```

バックアップ欄は、アプリ全体を復元するための場所です。
教材JSONを貼る場所ではありません。

---

## 12. バックアップのやり方

ブラウザ保存は消えることがあります。
**3日に1回はバックアップ**します。

おすすめは、

```text
⚙️ データ
↓
💾 バックアップ
↓
📥 ファイルとしてダウンロード保存
```

保存されるファイル名は、

```text
jukenquest_backup_日付.json
```

のようになります。

## 13. 教材作成プロンプト最新版

教材を作るときは、以下をChatGPTやClaudeに貼ります。

```text
あなたは中学受験の教材編集者です。以下の教材内容を、学習アプリ用の
「教材JSON(juken-pack@1)」に変換してください。

# 出力ルール(厳守)
- JSONのみを出力。前置き・説明・マークダウンの```は不要
- schema は "juken-pack@1" 固定。id は "pack_" + 教科略号(m/j/r/s) + 学年 + 内容
- subject は「算数」「国語」「理科」「社会」のいずれか

# ユニット構成(この順で)
1. INPUT: 重要知識のカード(front=問いかけ, back=答え。10〜20枚)
2. TRAIN: 練習問題 8〜15問。qtype をバランスよく混ぜる:
   - choice(4択): 5割程度
   - truefalse(○×): 2割程度。「まぎらわしい言い換え」で出す
   - multi(すべて選べ): 1〜2問。分類問題に
   - order(並べかえ): 1〜2問。年代順・手順・大小順があれば必ず入れる
   - flashcard(暗記カード): 1〜2問。「声に出して言おう」と促す文言で
   - input(記入): 最大2問まで。計算結果の数字など必要な時だけ
3. TEST: 本番形式 5〜10問(timeLimit=60、choiceとinput中心、hints不要)
4. BOSS: 総まとめ4〜6問(重要問題の類題、bossName付き)
5. WRITE: 手書きが必要な内容があれば paperTask に instructions と checkPoints(3〜4個)

# 錯乱肢ルール(最重要)
選択肢の誤答は、無関係な語ではなく「本当に覚えていないと選べない」ものにする。
各choice/multi問題の誤答には、次のいずれかを必ず1つ以上含めること:
- 同じカテゴリの紛らわしい別項目
- ありがちな計算ミス・勘違いの結果
- 一字違い・同音・似た言い回し
- 逆の意味・反対の操作
消去法で当たる「明らかに変な選択肢」は禁止。
truefalseも「惜しい言い換え」で作る。

# 問題フィールドの書き方
- choice: answer は正解の位置
- multi: answers は正解位置の配列
- order: items に正しい順で書く
- flashcard: front / back
- input: 別解は accept に入れる。acceptedAnswers は使わない
- 各問に explanation を必ず入れる
- TRAINの問題には hints を2段階入れる
- tag は「大分類:小分類」形式で統一
- level は 1(基本)〜5(難問)

# 対象・難易度
- 対象学年: 【小◯年】
- 難易度の目安: 【組分け・合不合・全統小・◯◯中過去問 など】

# 教材内容
【ここに教材テキスト・写真の書き起こし・範囲表を貼る】
```

## 14. よくあるエラーと直し方

| 症状 | 原因 | 直し方 |
|---|---|---|
| 教材追加欄が出ない | play版になっている | custom設定JSONを貼る |
| 登録できない | JSONの形が壊れている | ChatGPTに「このJSONを修正して」と頼む |
| `schema が必要です` と出る | schemaがない | `"schema":"juken-pack@1"` を入れる |
| 教材が上書きされた | 同じidを使った | 改訂ならOK。別教材ならidを変える |
| 記入問題の別解が効かない | `acceptedAnswers` を使っている | `accept` に直す |
| クエストに出てこない | subjectが違う/未登録 | データ画面で登録完了表示を確認 |
| 成績を消したい | 全データ初期化してしまいがち | まずバックアップを取ってから操作 |

## 15. 販売・配布時の注意

家庭内で使う場合と、販売する場合では注意点が違います。

### 家庭内利用

市販教材の内容をもとに、自宅学習用のクイズに変換して使う運用を想定します。

### 販売する場合

市販教材・過去問の本文をそのまま入れた教材JSONを販売してはいけません。
販売用は、オリジナル問題として作る必要があります。

## 16. 初心者向け・毎週の運用テンプレ

### 週テスト前

1. 範囲表または「まとめてみよう」を写真で用意
2. ChatGPTに教材JSONを作らせる
3. JSONをアプリに貼る
4. 子どもがTRAINを解く
5. 間違えたタグを親画面で見る
6. TESTとBOSSをやる
7. 白地図・漢字・計算過程はWRITEで紙に書く
8. バックアップする

### 組分け前

1. 第◯回〜第◯回をまとめた総復習パックを作る
2. INPUTで知識確認
3. TRAINで穴を見つける
4. REVIEWで弱点を潰す
5. TESTで本番確認
6. BOSSで仕上げ

## 18. 最終結論

このアプリは、コードを書く人向けではなく、

> **教材をJSON化して、貼って、遊ぶように反復する家庭学習OS**

として運用します。

初心者が覚えるべきことは3つだけです。

1. 教材は **教材JSON** にする
2. 教材JSONは **データ画面の「JSONを貼り付けて登録」** に入れる
3. 3日に1回は **バックアップ** する
